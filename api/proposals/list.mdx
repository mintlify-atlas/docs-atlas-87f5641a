---
title: "List Proposals"
api: "GET /api/proposals"
description: "Retrieve all proposals for the authenticated user, sorted by most recent first."
---

## Authentication

This endpoint requires authentication via Bearer token.

```bash
Authorization: Bearer <token>
```

## Response

<ResponseField name="proposals" type="array">
  Array of proposal objects belonging to the authenticated user.
  
  <Expandable title="Proposal Object">
    <ResponseField name="id" type="string">
      Unique proposal identifier (UUID).
    </ResponseField>
    
    <ResponseField name="proposalCode" type="string">
      Human-readable proposal code (format: PROP-YYYYMMDD-XXXX).
    </ResponseField>
    
    <ResponseField name="title" type="string">
      The title of the proposal.
    </ResponseField>
    
    <ResponseField name="description" type="string">
      The description of the proposal.
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Current status of the proposal (e.g., "draft", "submitted", "completed").
    </ResponseField>
    
    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp of when the proposal was created.
    </ResponseField>
    
    <ResponseField name="updated_at" type="string">
      ISO 8601 timestamp of the last update.
    </ResponseField>
    
    <ResponseField name="user_id" type="string">
      The ID of the user who owns the proposal.
    </ResponseField>
    
    <ResponseField name="user_email" type="string">
      The email of the user who owns the proposal.
    </ResponseField>
    
    <ResponseField name="user_name" type="string">
      The name of the user who owns the proposal.
    </ResponseField>
    
    <ResponseField name="uploaded_files" type="object">
      Dictionary of uploaded files organized by category (e.g., "rfp-document", "concept-document", "reference-proposals").
    </ResponseField>
    
    <ResponseField name="text_inputs" type="object">
      Dictionary of text inputs provided by the user.
    </ResponseField>
    
    <ResponseField name="metadata" type="object">
      Additional metadata including analysis results, status flags, and processing information.
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

```bash cURL
curl -X GET https://api.igadregion.org/api/proposals \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes

- **Sorting**: Proposals are returned in reverse chronological order (most recent first) based on the `created_at` timestamp.
- **User Filtering**: Only proposals belonging to the authenticated user are returned.
- **DynamoDB Keys**: Internal DynamoDB keys (`PK`, `SK`, `GSI1PK`, `GSI1SK`) are filtered out from the response.
- **Status Values**: Common status values include `"draft"`, `"submitted"`, `"completed"`, and `"archived"`.
